# 抖音带货达人采集工具

搜索抖音话题下的视频带货达人，自动化完成 **搜索 → 提取作者 → 扫描主页 → 分析带货 → 导出 Excel** 全流程。

## 功能概览

| 功能 | 说明 |
|------|------|
| 🔍 话题搜索 | 输入关键词（如"防晒"、"美妆"），自动搜索抖音相关视频 |
| 👤 达人提取 | 从搜索结果中提取视频作者，按粉丝数过滤，自动去重 |
| 📊 带货分析 | 扫描每位达人的主页作品，检测是否挂载小黄车/橱窗 |
| 📥 Excel 导出 | 输出结构化报表：达人链接、昵称、带货视频链接、最新发布时间 |
| ⚡ 并发加速 | 支持多达人同时扫描，3 档速度模式可调 |
| 🔐 多方式登录 | 浏览器提取 / rookiepy / CDP / 剪贴板 四种 Cookie 获取方式 |

## 快速开始

### 推荐方式：GUI 界面

```bash
python main.py --gui
```

启动后按照以下步骤操作：

1. **获取 Cookie**：点击右上角下拉框选择「登录提取 (推荐)」→ 点击「🔐 提取」→ 在弹出的浏览器中登录抖音
2. **输入关键词**：在「话题关键词」输入框中输入搜索词（如 `防晒`）
3. **调整参数**（可选）：扫描作品数、排序方式、发布时间等
4. **点击「🚀 开始采集」**：等待自动完成，导出按钮会自动启用

### 命令行方式

```bash
# 单次搜索
python main.py -k "防晒"

# 自定义参数
python main.py -k "美妆" -n 10 -s 1 -t 180

# 交互式（会提示输入关键词）
python main.py
```

### Cookie 状态栏

首次启动时，Cookie 栏会显示 `⚠️ 未设置`。有以下几种获取方式：

| 方式 | 操作 | 适用场景 |
|------|------|---------|
| **登录提取 (推荐)** | 下拉选此项 → 点「🔐提取」→ 弹出浏览器登录 | 首次使用，最可靠 |
| **rookiepy 提取** | 下拉选此项 → 点「🔐提取」 | 浏览器已登录，需管理员权限 |
| **CDP 提取 (备用)** | 下拉选此项 → 点「🔐提取」 | 前两种失败时的备选 |
| **📋 粘贴** | 从浏览器 F12 复制 cURL → 点此按钮 | 手动获取，最精确 |

> 登录成功后 Cookie 自动保存到 `.env` 文件，下次启动无需重新登录。

### 采集配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 话题关键词 | （必填） | 抖音搜索词，如 `防晒`、`美妆`、`穿搭` |
| 扫描作品数/人 | 10 | 每位达人拉取最近 N 条作品判断是否带货。越大越准但越慢 |
| 排序方式 | 最多点赞 | 综合排序 / 最多点赞 / 最新发布 |
| 发布时间 | 半年内 | 不限 / 一周内 / 半年内 |
| 搜索上限 | 200 | 搜索结果视频总条数。抖音 API 单次最多约 500 条 |
| 最低粉丝数 | 0 | 低于此值的作者直接跳过。0 = 不过滤 |
| 并发数 | 3 | 同时扫描的达人数（见速度指南） |
| 速度模式 | 安全 | 安全 / 快速 / 极速（见速度指南） |
| 多轮搜索 | 关闭 | 用点赞 + 最新两种排序各搜一轮，合并去重。可多获 30%~80% 达人 |

### 结果表格

| 列 | 说明 |
|----|------|
| # | 序号 |
| 昵称 | 达人抖音昵称 |

| 视频/挂车 | 格式：`视频总数/挂车数量` |

- **双击 ✓ 行**：在浏览器打开该达人主页

### 进度条

采集过程分为 4 个阶段：

| 进度 | 阶段 | 说明 |
|------|------|------|
| 0% → 15% | 搜索中 | 调用抖音搜索 API，分页拉取视频列表 |

## Cookie 配置详解

### 方法一：GUI 登录提取（推荐）

1. 在 Cookie 栏下拉选择「登录提取 (推荐)」
2. 点击「🔐 提取」按钮
3. 在弹出的 Chromium 窗口中扫码或账号登录抖音
4. 登录成功后程序自动检测并保存 Cookie，浏览器窗口自动关闭

> Cookie 保存在 `main/.env` 文件中，下次启动自动读取。

### 方法二：手动复制（最稳定）

1. 用 Chrome/Edge 打开 [douyin.com](https://www.douyin.com/) 并登录
2. 按 `F12` 打开开发者工具 → 切换到 **Network（网络）** 标签
3. 刷新页面（`F5`）
4. 在左侧列表中**右键任意请求** → **Copy** → **Copy as cURL (bash)**
5. 回到程序 → 点击「📋 粘贴」按钮
6. 程序会自动从剪贴板提取 Cookie 并保存

### 方法三：直接编辑 .env 文件

在 `main/.env` 中填写：

```
DY_COOKIE='你的完整Cookie字符串'
```

获取完整 Cookie 字符串的方法：浏览器 F12 → Application → Cookies → douyin.com → 逐个复制 name=value 用 `; ` 拼接。

### Cookie 有效性判断

程序通过检测以下关键字段判断 Cookie 是否有效：

- `sessionid` / `sessionid_ss` — 用户登录会话
- `sid_guard` — 会话守护

> 只有包含以上任一字段的 Cookie 才会被认为是"已登录"状态。

## 速度优化指南

### 速度模式对比

| 模式 | 搜索间隔 | 达人间隔 | 作品间隔 | 风控风险 | 适用场景 |
|------|---------|---------|---------|---------|---------|
| **安全** | 3~5s | 3~8s | 1~3s | 低 | 首次使用、长时间无人值守 |
| **快速** | 1.5~3s | 2~4s | 0.8~1.5s | 中 | 日常采集 |
| **极速** | 0.8~1.5s | 1~2.5s | 0.3~0.8s | 高 | 少量达人、需要最快速度 |

> 极速模式下会自动跳过作品详情 API 检测，挂车判断仅依赖主页列表数据，可能对部分"隐式挂车"漏判。

### 并发数建议

| 速度模式 | 建议并发数 | 最大安全值 |
|---------|-----------|----------|
| 安全 | 2 ~ 4 | 4 |
| 快速 | 3 ~ 6 | 6 |
| 极速 | 5 ~ 8 | 8 |

> **风控警告**：并发数过高 + 极速模式可能触发抖音限流（HTTP 444），程序会自动重试但会显著变慢。建议先用安全模式试探，逐渐调高。

### 多轮搜索

勾选「多轮搜索」复选框后，程序会用「最多点赞」和「最新发布」两种排序各搜索一轮，合并去重。**可比单轮多获取 30%~80% 的达人和视频**，但搜索时间翻倍。

建议搭配「极速模式」使用，在速度和多获取之间取得平衡。

### 典型耗时估算（搜索 200 条 / 扫描 10 条作品 / 50 位达人）

| 配置 | 预计耗时 |
|------|---------|
| 安全 + 单轮 + 并发 3 | ~3.5 分钟 |
| 快速 + 单轮 + 并发 5 | ~1.2 分钟 |
| 极速 + 多轮 + 并发 8 | ~40 秒 |
| 极速 + 单轮 + 并发 8 | ~30 秒 |

## 常见问题

### Q: 启动后 Cookie 栏显示「未设置」怎么办？

点击「登录提取 (推荐)」→「🔐 提取」，在弹出的浏览器中完成抖音登录即可。

### Q: 搜索不到任何视频？

1. 检查 Cookie 是否有效（看状态栏是否显示 ✅）
2. 尝试换一个更通用的关键词
3. 确认抖音账号未被风控（尝试在浏览器中搜索同样的关键词）

### Q: 很多达人显示「无作品」？

可能是达人设置了隐私保护，或账号已被注销/封禁。这是正常现象。

### Q: 采集过程中出现 HTTP 444 错误？

这是抖音的限流措施。程序会自动重试，但如果频繁出现：
1. 降低速度模式（改为「安全」）
2. 降低并发数
3. 等待几分钟后重试

### Q: 为什么有些明显带货的达人被判为「非带货」？

两条可能原因：
1. 该达人最近的 N 条作品（默认 10 条）恰好没有挂车
2. 极速模式下跳过了详情 API，主页列表中的挂车标记未显示

解决：增加「扫描作品数」或关闭极速模式。

### Q: 「登录提取」弹出了浏览器但没反应？

1. 确保已运行 `playwright install chromium`
2. 尝试切换到「CDP 提取 (备用)」方式
3. 或使用「📋 粘贴」手动获取 Cookie

### Q: 点击停止后仍在运行？

停止后正在执行中的请求会继续完成（通常 1~3 秒），之后程序会自动停止并导出已收集的结果。
